Garbage Collection

21 December 2016

When an object is created it takes up memory. Memory is not infinite and the more that is used up, the less there is for allocation. This is why memory that is no longer being used must be cleaned up so it can be re-used.

Garbage collection looks for things stored in memory that is no longer necessary and frees that space. There is a distinction between Garbage Collection and Compaction in which allocated objects are moved together in the memory and empty spaces are left together. Object creation and memory allocation can get much more complex, and the basics of these require a better understanding of the stack and heap, which I will write up on in another post.
