Finding The Pattern

25 September 2015

One of the things I am trying to get better at is writing concise code and a great way to do this is by reducing duplication. Initially I find that I write a lot of if statements, as these are the closest to real language that I use when describing the problem or visualizing it for the first time.

Sometimes this duplication is more subtle than I would like and an alorithm is hiding in there somewhere. It is more difficult to find once you have started to write, but again I think an old pen and paper helps. Sometimes, it takes a little time to find the pattern and additional tests can help to find the common patterns.
