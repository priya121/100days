  1. do-while loops

    22 September 2015

    While and Do While loops are two of the most common loops and they exist across most languages. A normal while loop executes the code inside it provided the boolean expression it evaluates is true.

    It is easy to enter a never ending while loop accidently if you forget to include a terminating condition. So it is important to always consider if the condition that you want to exit the loop on is ever met. If it is not your program will never exit the loop or go past that point.

    eg.

    while (true) 
    do (something)
 until (terminating condition)

    A do while loop however, is that the code is always executed at least once, because the condition or expression is evaluated at the bottom of the loop.

    This is an advantage in certain circumstances eg. if you need to make sure an output is printed at least once even if the condition within the while loop is not true.

    So next time you want to use a while loop, it is worth thinking about if it would make more sense to use a do while loop.

  2. Dotfiles

    24 August 2015

    Setting up a new machine can take time and it is always nice to take your time customizing the settings just how you like them. But this can take a long time if there are many settings you have to set manually. A useful way to speed up this process and cut down duplication time is to copy across your dotfiles from git.

    I recently did this and now am able to quickly work on my projects on the new machine just as I did before. In order to clone across the files, it is also necessary to set up your new machine with an SSH key on git. There are many resources that help with this. One I used is: [https://help.github.com/articles/generating-ssh-keys/] and it is a simple process.

    Now I can manage my settings across multiple devices and am more comfortable with adding to my dotfiles. Next step is to change my vim colours!

  3. Encapsulation in Ruby

    26 August 2015

    There are many useful components in Ruby as a language and one I looked into closer recently is Encapsulation.

    Encapsulation reduces coupling as it hides the internal representation or data structure of a particular object. This makes it easier to make changes when they become necessary, as if there are changes that need to be made to the data within the class, it will handle these changes and they will not have to be made throughout the entire program or application.

    Encapsulation makes more sense when programs get larger, but I am going to try and implement it as soon as possible and this will make things easier to change.

  4. Enums in Java

    30 August 2015

    Enums in Java allow for a specific data type to be set as a list of predefined constants. Most of the time, these predefined constants represent constants in the real world, like days of the week or months in a year. These rarely change so can be set for the duration of a program and do not represent numeric values or textual data.

    Enums in Java are type safe, meaning that they cannot be reassigned to another type once they have been set. It is also possible to set values for the enums and retrieve these by calling .values() on them.

    They can be used somewhat recklessly when defining a type on a set of values, but used well can be useful.

  5. Throwing Exceptions

    02 September 2015

    Throwing an exception is a way of handling an error that a method may produce for a specific reason that was expected by the programmer who wrote it.

    In java a complier will not let you compile a method if an exception is written that has not been handled or specifically declared.

    Even though it is possible to declare that the main method throws the exception, it is always nicer to pass the declaration to another method and print a message to the user after another method has caught the exception. It is also good practice to tell the user how to address the error if at all possible.
