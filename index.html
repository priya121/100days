  1. do-while loops

    24 September 2015

    While and Do While loops are two of the most common loops and they exist across most languages. A normal while loop executes the code inside it provided the boolean expression it evaluates is true.

    It is easy to enter a never ending while loop accidently if you forget to include a terminating condition. So it is important to always consider if the condition that you want to exit the loop on is ever met. If it is not your program will never exit the loop or go past that point.

    eg.

    while (true) 
    do (something)
 until (terminating condition)

    A do while loop however, is that the code is always executed at least once, because the condition or expression is evaluated at the bottom of the loop.

    This is an advantage in certain circumstances eg. if you need to make sure an output is printed at least once even if the condition within the while loop is not true.

    So next time you want to use a while loop, it is worth thinking about if it would make more sense to use a do while loop.

  2. Counting in Binary

    22 September 2015

    Mathematics is not vital to knowing how to program, but it definitely helps if you are good at it. This is formal logic. If x then y is not the same as if y then x (a mistake people make all the time in everyday conversation). Counting in binary is simple if you imagine that all a computer has access to is 0 and 1 or on and off in an electrical circuit.

    Because of this everything it reads has to be repreeneted in 0s and 1s. So this means that once it has used 0 and 1 to represent a number it can repetitively reuse 0 and 1 to add onto the end of a previous combination of 0 and 1.

    This is best explained if you imagine that you can only count using the digits 0–10 and a–j once you have counted to 10 using the numbers in order to carry on 0a would represent 11 and 1a would be 21 etc.

    In a similar way a computer uses 0s and 1s and combinations of these when it needs to add to a previous value.

    The easiest way to calculate the binary value of a number is to divide a number by 2 and record the remainder value as part of the binary result until the number you are dividing is one.

  3. Stacks

    22 September 2015

    Stacks are a pretty fundamental concept in CS. They represent how data is allocated to memory at a basic level. A stack area on computer memory is of a fixed origin and variable size. The two basic operations that are used most frequently on a stack is inserting an item and removing an item and a stack uses the concept of last in first out (LIFO). In Java it is possible to create a Stack and call certain predefined methods on it like push, pop, peek, search and size. Like most Java lists a stack has to contain matching objects within it that must be declared when it is first created.

  4. Method Signatures in Java

    22 September 2015

    The main class in java must be present in the application at least once for it to be able to run and has a pretty standard signature.

    Methods in java have to have certain elements in order for the Java package to be able to collect the parts of the program togethr and process them in a logical way.

    Firstly a method must be public or private. Public means that it is available across packages whereas private means it is only accessible within the current package. Then if a method is static(that is, it is not called by Objects within the class) this must also be part of the method signature.

    These are followed by what the method signature will return (String, Integer, String[], void etc.) and the method name.

    In Java if a method takes an arguement, the type of arguement must also be declared within parentheses, this is true even if the method takes no arguement with the parentheses being left blank.

    Method signatures in Java seem complicated at first, but I have been told that this makes things clearer in the long term, so far most of my programs and tasks have been small, but I will report back when I know if this is true.

  5. Enums in Java

    22 September 2015

    Enums in Java allow for a specific data type to be set as a list of predefined constants. Most of the time, these predefined constants represent constants in the real world, like days of the week or months in a year. These rarely change so can be set for the duration of a program and do not represent numeric values or textual data.

    Enums in Java are type safe, meaning that they cannot be reassigned to another type once they have been set. It is also possible to set values for the enums and retrieve these by calling .values() on them.

    They can be used somewhat recklessly when defining a type on a set of values, but used well can be useful.
