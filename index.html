  1. static variables

    07 January 2016

    A static method or vaiable is one that does not belong to a particular object but to a class. There are a few reasons for doing this. You may want to keep track of how many objects of a certain type ofe class are created, or assign an id to each object created or just access a variable from outside the class. It is possible to then access the static, constant from outside the class, but in order to make it unchangable it is necessary to also declare it final.

    It is convention to write constants in capitals. They will still work as expected if this is not done, but doing this means that it is clear to anyone reading your code that the variable is a constant.

    I have tended to avoid using constants, but I am going to try and introduce them wherever necessary in my Java TicTacToe.

  3. Memory Management in Java

    04 January 2016

    Memory is something that seems pretty abstract before you face a real problem with it. Objects and data are stored on the Java Virtual Machine and when writing simple small programs, when and how these are created is not that important.

    It becomes more important when lots of different classes interact and the order objects are created and destroyed affects the efficiency of a program.

    Java has two areas of memory. The stack and the heap.

    The stack is where variables live and they live on the stack frame where the method they belong to is stored. Object reference variables also are stored in this way.

    All objects however, live on the heap so that they can be destroyed by the garbage collector when they are no longer needed (when there are no longer any live references to them). If your program starts to run low on memory, the garbage collector destroys the objects that are no longer needed and memory will not run out.

    I am working on a longer blog post about testing in Vim for Java using Gradle and even though I’m sure I’ve just scratched the surface, hopefully understanding memory management will make understanding how threads and sockets work more clear.

  4. Garbage Collection

    21 December 2015

    When an object is created it takes up memory. Memory is not infinite and the more that is used up, the less there is for allocation. This is why memory that is no longer being used must be cleaned up so it can be re-used.

    Garbage collection looks for things stored in memory that is no longer necessary and frees that space. There is a distinction between Garbage Collection and Compaction in which allocated objects are moved together in the memory and empty spaces are left together. Object creation and memory allocation can get much more complex, and the basics of these require a better understanding of the stack and heap, which I will write up on in another post.

  5. Arduino

    18 December 2015

    Today Micah, the founder of 8th Light held his second day of the mastery cohort and all of us were kindly given an Arduino starter kit. I had wanted one for a while and it felt like an early Christmas present.

    Arduino is a company that designs, manufactures and sells micro-contollers and micro-controller accessories. These allow you to connect to your computer and program various circuits and components. It was interesting to read through the introductory book that came with the starter kit and it reminded me of my GCSE Physics classes.

    The first exercise was to build a simple series circuit and write up code that would cause a LED bulb to light up. We then connected a paralell circuit and made flashing lights that were supposed to look like a spaceship. It was cool to see how simple code would cause the electrical components to do what we wanted them to do. Well, most of the time anyway!

    After lunch we attempted a Love Machine exercise that used a thermometer component to tell how “hot” you were, and also managed to get the keyboard exercise working with 3 simple musical notes.

    It was a great introduction to how software connects to simple hardware and I am excited to try some more of the exercises out in my own time. One I have my eye on is the Crystal Ball, which looks more complex as there are many more connections that must align with specific parts of the digital display. Hopefully, I’ll manage to get it working and will be able to build on it once I do.
