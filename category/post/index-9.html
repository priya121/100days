  2. Enums in Java

    22 October 2015

    Enums in Java allow for a specific data type to be set as a list of predefined constants. Most of the time, these predefined constants represent constants in the real world, like days of the week or months in a year. These rarely change so can be set for the duration of a program and do not represent numeric values or textual data.

    Enums in Java are type safe, meaning that they cannot be reassigned to another type once they have been set. It is also possible to set values for the enums and retrieve these by calling .values() on them.

    They can be used somewhat recklessly when defining a type on a set of values, but used well they can be useful.

  3. Testing I/O

    23 October 2015

    Testing a randomised output can be difficult, as if an actual random number is returned, the result will be different every time. In order to tackle this I have used interfaces that swap in for certain tests and then also allow for simulated input to be tested.

    In Java the simplest way to test input and output is to use a ByteArrayInputStream and ByteArrayOutputStream. These can then be easily swapped out for System.in and System.out when used in the main method and the program is run.

  4. Java IO Streams

    26 October 2015

    Java Input Output streams abstract different ways in which you can input and output data. The stream can be sent to or taken from any source, it does not matter. It is really a stream or flow of bytes from one destination to another.

    Java divides streams into bytes or character based categories. The character unit of a character based stream is a Unicode or ASCII character and transfers input and output of 8-bit bytes at a time.

    This is in contrast to FileReader and FileWriter, which even though they internally use FileInputStream and FileOutputStream, read and write two bytes at a time. This can slow down your program and so it can be an advantage to use FileInputStream and FileOutputStream directly to transfer the data across sources.

  5. Recursion

    27 October 2015

    Recursion is one of the most useful things I learnt when I first began how to code.

    Recursion is simply when a method or function calls itself with changing data being assessed.

    It is a good example of how a simple algorithm can handle complexity by analyzing data that changes. It is similar to a loop, but instead of being given a finite termination condition or number of times to execute, it is able to assess itself when the condition has been met and this can be an extremely powerful tool.
