  1. Enums in Java

    22 September 2015

    Enums in Java allow for a specific data type to be set as a list of predefined constants. Most of the time, these predefined constants represent constants in the real world, like days of the week or months in a year. These rarely change so can be set for the duration of a program and do not represent numeric values or textual data.

    Enums in Java are type safe, meaning that they cannot be reassigned to another type once they have been set. It is also possible to set values for the enums and retrieve these by calling .values() on them.

    They can be used somewhat recklessly when defining a type on a set of values, but used well can be useful.

  2. Throwing Exceptions

    22 September 2015

    Throwing an exception is a way of handling an error that a method may produce for a specific reason that was expected by the programmer who wrote it.

    In java a complier will not let you compile a method if an exception is written that has not been handled or specifically declared.

    Even though it is possible to declare that the main method throws the exception, it is always nicer to pass the declaration to another method and print a message to the user after another method has caught the exception. It is also good practice to tell the user how to address the error if at all possible.

  4. Input/Output

    22 September 2015

    Everything has to be explicitly declared in Java. I am starting to get used to it, even though the format looks different, the general structure of a class is not too different to Ruby, even though there are a lot of restrictions on what you can and cannot do.

    In order to use Input Output in Java, the simplest way to do this is the InputStreamReader and OutputStreamReader.

    The sytnax for these is System.in and System.out.print() respectively, but the format of these has to be changed accordingly when a certain type of display is needed.

    It is also possible to use the Scanner class that parses primitive types and strings suing regular expressions.

    There are so many tools and libraries that you can import in java that help you, it can be difficult to figure out which ones are best to use, but I am sure this is something that comes with practice and familiarity with the language.

  5. Interfaces

    22 September 2015

    Interfaces are a set of classes that used to create an abstraction between two other related classes. They do not contain any new code but hold behaviours and method signatures and should be abstract classes with abstract names.

    The advantage of using interfaces is that they help eliminate unnecessary dependencies that create rigid and fragile software. Adding interfaces hides the implementation of certain classes from others when they are not needed and allows for easier changes to the code later down the line.

    Ruby and other dynamic languages do not need to have an explicit interface class, but it is necessary to define an interface class type explicitly for statically typed languages like Java.
