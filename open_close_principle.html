Open Close Principle

29 September 2015

I am still getting familiar with the SOLID principles and am trying to assess their value. It is often only when I see them in bad code that I have written or problems that I come up against after not implementing them that I can spot them.

But I think reading through them at this stage is a good exercise. The Open Close Principle is the next one I am looking at, which states, that software entities should be closed for modification but open for extension.

That is, that it should be possible to make changes to the exisiting code and extend its behaviour without breaking the exisiting functionality of the code. Interfaces help with this as they allow you to extend the code more easily and help create compact classes that hold the required behaviour.

I am using this principle to a certain degree without realising it, but will try and be more concious of when I am using it and make sure I do so when required.
